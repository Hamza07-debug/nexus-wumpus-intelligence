Knowledge Base (KB)

The agent maintains a set of propositional logic clauses in Conjunctive Normal Form (CNF). Each clause is a disjunction of literals — for example, (¬TRAP₁,₂ ∨ WIND₁,₁) states "if cell (1,2) has a trap, then cell (1,1) has wind."

Percept Encoding

When the agent visits a cell, sensor readings (breeze, stench, glitter) are converted into logical axioms:

  • Breeze ↔ Adjacent Pit: A breeze at cell (r,c) means at least one neighbor has a pit, encoded as a biconditional in CNF.
  • Stench ↔ Adjacent Wumpus: Same logic applies for stench and wumpus presence.

Resolution Refutation

To determine if a cell is safe, the engine uses proof by contradiction:

  1. Negate the query (e.g., assume cell has a trap).
  2. Add this negated hypothesis to a Set of Support (SOS).
  3. Repeatedly resolve SOS clauses against KB clauses.
  4. If an empty clause (⊥) is derived → the original query is proven true.
  5. If no empty clause is found within the step limit → result is inconclusive.

Decision Loop

Each step: infer safe/hazard cells → plan a path via BFS over confirmed-safe cells → move to the most informative unexplored safe cell. If stuck, the agent may fire an arrow toward a suspected Wumpus.
